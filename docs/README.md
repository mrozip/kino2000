## Documentation Guidelines

- Keep stable product and architecture decisions in Markdown.
- Keep active backlog items in GitHub Issues and GitHub Projects.
- Do not duplicate the full backlog in Markdown and GitHub Issues.
- Use relative Markdown links so the docs work in GitHub and Obsidian.
- Prefer focused files over large catch-all planning documents.
- Add Architecture Decision Records when a decision needs context, options, and consequences.
