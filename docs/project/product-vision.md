# Product Vision

kino2000 is an online movie database where users can search for films, record ratings, add favourites, maintain a profile, and eventually interact with other users.

The project is intended as a practical DevOps and cloud development learning exercise. It should demonstrate good infrastructure, CI/CD, security, documentation, and repository-management practices without introducing unnecessary complexity.

## Project Goals

- Build a functional movie-tracking web application.
- Follow good CI/CD and infrastructure-as-code practices.
- Use GitHub for source control, project planning, issue tracking, and delivery workflows.
- Maintain project documentation in Markdown.
- Keep the application architecture simple and modular.
- Minimise Azure cost and operational overhead.
- Build the application incrementally through clearly defined sprints.

## Scope Controls

The following should remain outside the initial project scope unless a clear requirement emerges:

- Microservices
- Kubernetes
- Azure Cache for Redis
- Event-driven messaging
- Multiple databases
- GraphQL
- Custom password authentication
- Recommendation algorithms
- Real-time chat
- Direct messaging
- Multiple permanent Azure environments
- Complex moderation systems
- Container hosting solely for demonstration purposes

The project should prioritise a small, maintainable application that demonstrates strong engineering and DevOps practices over unnecessary platform complexity.
