# Project Management

Use GitHub Issues and GitHub Projects as the main source of truth for active work.

## Issue Types

Recommended issue types:

- Feature
- Bug
- Technical task
- Documentation
- Research spike

## Project Fields

Recommended project fields:

- Status
- Sprint
- Priority
- Type
- Area
- Estimate
- Environment

Suggested statuses:

- Backlog
- Ready
- In progress
- In review
- Done

## Issue Content

GitHub Issues should contain:

- User story or technical objective
- Acceptance criteria
- Technical notes
- Testing requirements
- Dependencies
- Links to related pull requests

## Markdown Documentation

Markdown documentation should contain:

- Product vision
- Architecture
- Roadmap
- Sprint goals
- Sprint reviews
- Retrospectives
- Architecture Decision Records
- Deployment instructions
- Troubleshooting documentation

The full backlog should not be duplicated in both Markdown and GitHub Issues.
