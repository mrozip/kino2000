## Delivery Principles

- Keep the application architecture simple and modular.
- Minimise Azure cost and operational overhead.
- Prefer working vertical slices over broad incomplete foundations.
- Keep the active backlog in GitHub Issues and GitHub Projects.
- Use sprint documents for goals, scope, and acceptance criteria.
